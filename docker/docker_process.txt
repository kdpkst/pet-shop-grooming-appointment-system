(1) skip test to package the application into the jar package (automatically placed in "target" folder):
mvn package -DskipTests=true

(2) write a Dockerfile to build the jar package in a container  
commands used: $ code Dockerfile

(3) write docker-compose.yml to connect our container and MySQL container
commands used: $ docker-compose up

(4) put the jar package, Dockerfile and docker-compose.yml in the same directory

(5) navigate into that directory, open windows powershell (shift+right click)
input a command: $ docker-compose up

(6) on the localhost, push image that has been built to my own docker hub repo:
$ docker login
<enter user name and password for Docker Hub Repository>

$ docker tag first-image {docker-hub-username}/{default-repo-folder-name}:first-image
e.g. docker tag dockerbuild-springboot-app jammiechen/pet-grooming-shop:dockerbuild-springboot-app

$ docker push {docker-hub-username}/{default-repo-folder-name}:first-image
docker push jammiechen/pet-grooming-shop:dockerbuild-springboot-app 

(7) on the server, login account. then pull image from my docker hub repo: 
$ docker pull {docker-hub-username}/{repo-name}:image-name
e.g., 
$ docker pull jammiechen/pet-grooming-shop:dockerbuild-springboot-app
